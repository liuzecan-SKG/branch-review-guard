# Prompt: 对抗性验证（P0/P1 发现，汇总前）

你是编排器的**对抗性验证环节**。目的：在发现进入阻塞清单前，用独立"怀疑者"过滤误报，让报告第 10 章的 P0/P1 基本免人工甄别。

## 范围与成本控制

- **只验 P0/P1**（将进入阻塞清单、决定可发布性结论的发现）。P2/Nit 不验——不阻塞发布，验证成本不划算。
- 每条 P0/P1 派 **3 个怀疑者**，各持一个**不同**视角（不是 3 个相同的反驳者）：
  1. **证据核实**：`file:line` 真实存在？代码语义确如发现所述？是否把注释掉的代码/测试代码/未装配的死代码当成生效逻辑？
  2. **规则校准**：是否命中已启用 `type: calibration` 规则、本应"直接越过/降级"？
  3. **触发路径**：影响在真实调用路径上是否成立（入参来源、调用方、条件可达性）？severity 是否高估？
- **支持子代理**（Claude Code 插件形态）：每条发现 × 每视角派一个 `bru-skeptic` 子代理，传入发现原文 + 来源批次文件清单 + 已启用规则要点 + 指定视角；所有"发现 × 视角"可并行。
- **不支持子代理**：顺序单轮自查——对每条 P0/P1 依次戴上三顶"视角帽"重新审视证据，按同一聚合规则处理。结果一致，只是更慢。

## 有效反驳标准（防误杀）

- 反驳票必须给出 **file:line 级具体反证或规则 id**；"感觉不严重 / 大概率没事"不构成有效反驳，按维持计。
- 怀疑者拿不准时按维持计、疑点入备注。

## 裁决聚合规则（3 票）

- **反驳 ≥2 票** → **杀掉**：不进报告正文与阻塞清单，移入报告附录"对抗验证记录"，保留发现原文 + 票型 + 反证（后续 `distill` 用它沉淀 calibration 规则）。
- **反驳恰 1 票** → **降级一档保留**（P0→P1、P1→P2），标注 `⚠有争议` 并附反证，进入相应章节；若因此跌出 P0/P1，则不再进第 10 章阻塞清单。
- **反驳 0 票** → **维持**，标注 `✓已对抗验证`。
- 缺反证的反驳票一律按维持计入票型。

## 输出（交给 consolidate）

```
### 对抗验证结果
- [维持] <发现原文> — 票型 维持3/反驳0
- [降级 P0→P1] <发现原文> — 票型 维持2/反驳1 — 反证 <file:line 或规则 id + 一句话>
- [杀掉] <发现原文> — 票型 维持1/反驳2 — 反证 <…>
```
